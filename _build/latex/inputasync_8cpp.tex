Description \+:

This node asynchonrously captures user inputs and publishes them. Having an extra node that can handle inputs is very important, because inputs are generally blocking and it is not always feasible to interrupt everything until the user inputs something. To request user input, another node must simply publish any message to the \char`\"{}/ui\+\_\+inputrequest\char`\"{} topic, and the next user input will be published to the \char`\"{}/ui\+\_\+inputasync\char`\"{} topic.
